## Project Overview

**Hyperlocal Marketplace** is a full-stack e-commerce platform that connects local buyers with sellers in their ZIP code area. The platform supports three user roles: **Buyers**, **Sellers**, and **Admins**, each with dedicated dashboards and functionality.

### Table Descriptions

#### **Locations**
Stores ZIP code data with geolocation for hyperlocal filtering.

#### **Users**
Buyer accounts with profile information and location.

#### **Shops**
Seller accounts with shop details and location.

#### **Admins**
Platform administrators with full system access.

#### **Products**
Product catalog with pricing, inventory, and shop association.

#### **ProductImages**
Multiple images per product (one-to-many relationship).

#### **ProductReviews**
User reviews and ratings for products.

#### **ORDERS**
Complete order information stored as JSON for flexibility.

#### **CONVERSATION**
Chat rooms between buyers and sellers.

#### **MESSAGES**
Individual chat messages with sender identification.

## Setup Instructions

### Prerequisites
```bash
# Required Software
- Node.js (v16 or higher)
- MySQL (v8 or higher)
- npm or yarn
- Git
```

### Backend Setup

```bash
# 1. Navigate to backend directory
cd db2_test

# 2. Install dependencies
npm install

# 3. Create .env file in config directory
mkdir config
cd config
# Create .env file with the following:
```

**config/.env**
```env
# Database Configuration
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=hyperlocal_marketplace
DB_PORT=3306

# JWT Secret
JWT_SECRET=your_super_secret_jwt_key_change_this_in_production

# Stripe API Keys
STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key

# Server Port
PORT=3000

# CORS Origins
CORS_ORIGIN=http://localhost:5173
```

```bash
# 4. Set up MySQL database
mysql -u root -p

# Run the following in MySQL:
source DATABASE_SETUP.sql

# Or copy-paste the DATABASE_SETUP.sql contents

# 5. (Optional) Seed mock data
node seed_mock_data.js

# 6. Start the backend server
npm start
# Server runs on http://localhost:3000
```

## Troubleshooting

### Common Issues

#### 1. Database Connection Error
```
Error: ER_ACCESS_DENIED_ERROR
```
**Solution**: Check MySQL credentials in `config/.env`

#### 2. JWT Token Invalid
```
Error: jwt malformed
```
**Solution**: Ensure JWT_SECRET is set correctly

#### 3. Stripe Payment Fails
```
Error: No such PaymentIntent
```
**Solution**: Verify Stripe keys are correct (test/live mode)

#### 4. CORS Error
```
Access to fetch blocked by CORS policy
```
**Solution**: Add frontend URL to CORS origins in server.js

#### 5. Socket.IO Connection Failed
```
WebSocket connection failed
```
**Solution**: Check firewall settings and Socket.IO CORS config

#### 6. File Upload Error
```
Error: MulterError: Unexpected field
```
**Solution**: Verify field names match between frontend and multer config
